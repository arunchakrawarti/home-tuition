# Logical Grouping:

- primary for main branding colors.
- gray for neutral tones.
- success, danger, and warning for stateful feedback colors.
- Specific sections like meteorite or UI-Specific Colors for unique contexts.

# Consistent Naming:

- CamelCase for Tailwind integration (primaryLight, grayDarker, etc.).
- Avoids conflicts with default Tailwind CSS classes.

# Dynamic Adaptability:

- Switching themes dynamically is possible, as each variable is tied to a CSS variable like var(--theme-default-primary).
